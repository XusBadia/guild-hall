# Guild Hall Tileset Brief for PixelLab

## Goal
Create a cohesive **RPG guild hall room pack** for the Guild Hall app.

This product uses **two distinct visual languages**:
- **Agents tab:** large, polished character illustration portraits
- **Map tab:** small directional sprites walking inside handcrafted room scenes

This brief is only for the **Map tab environment art**.

## Overall art direction
- Style: **premium pixel-art RPG management sim**
- Camera: **top-down / slightly isometric** (consistent across all rooms)
- Tone: warm, magical, readable, game-like, production-quality
- Target feeling: halfway between a party base in a JRPG and a polished indie tactics game
- Rooms should feel **lived in**, not like generic placeholder tiles
- Prioritize **clarity of floor space and walkable lanes** so small sprites read well on top

## Technical constraints
- Final output should work as **room background scenes** for a web UI
- Rooms need clean silhouettes and readable props at small scale
- Keep the **central walk lane visually clear** in each room
- Avoid busy floor patterns where sprites patrol
- Leave enough negative space around spawn/patrol areas for 1–2 moving characters
- Lighting should support UI overlays and status badges

## Important product rule
**Do not draw large character portraits in these scenes.**
These rooms are backgrounds for the map view only.
Characters already exist as separate pixel sprites and large illustration art elsewhere.

## Shared quality bar
Every room should feel:
- like part of the **same guild complex**
- stylistically consistent with the existing Throne Room and Forge
- premium enough that a player would believe this is a real management RPG
- readable at a glance in a browser interface

## Deliverable preference
Best case:
- one polished room scene per location
- plus layered thinking that could later become reusable tiles/modules

If choosing between beauty and modularity for v1:
- prefer **beautiful, readable finished rooms**
- but structure props logically so future tile extraction is possible

## Summary
We need **7 missing guild room scenes** for a web-based RPG guild interface.
They must support **small walking sprites**, match an existing fantasy guild aesthetic, and feel like a serious product rather than placeholder concept art.
